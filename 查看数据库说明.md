# 查看 SQLite 数据库的方法

## ⚠️ VS Code SQLite 扩展问题

VS Code 的 SQLite 扩展有时会出现兼容性问题（如当前的 "Parse error near line 4: no such column: table" 错误）。

## ✅ 推荐方法

### 方法 1: 使用终端命令（最简单）

```bash
# 进入项目目录
cd /home/flowteam/zqy/agent-safe-probe-x

# 查看所有任务
sqlite3 frontend/tasks.db ".mode column" ".headers on" "SELECT * FROM tasks"

# 查看表结构
sqlite3 frontend/tasks.db ".schema"

# 进入交互式界面
sqlite3 frontend/tasks.db
```

### 方法 2: 使用 SQLite 浏览器工具

1. 安装 DB Browser for SQLite
   ```bash
   sudo snap install sqlitebrowser
   # 或者
   sudo apt-get install sqlitebrowser
   ```

2. 打开应用，选择 "打开数据库"，导航到：
   `/home/flowteam/zqy/agent-safe-probe-x/frontend/tasks.db`

3. 在 "浏览数据" 选项卡中查看数据

### 方法 3: 使用命令行脚本

创建以下脚本文件 `view_db.sh`：

```bash
#!/bin/bash
cd /home/flowteam/zqy/agent-safe-probe-x

echo "=== 任务列表 ==="
sqlite3 frontend/tasks.db -column -header "
SELECT 
    substr(id, 1, 12) as '任务ID',
    status as '状态',
    datetime(created_time) as '创建时间',
    progress as '进度%'
FROM tasks 
ORDER BY created_time DESC
LIMIT 10"

echo -e "\n=== 任务状态统计 ==="
sqlite3 frontend/tasks.db -column -header "
SELECT 
    status as '状态',
    COUNT(*) as '数量'
FROM tasks 
GROUP BY status"

echo -e "\n输入任务ID查看详情，或按 Enter 退出: "
read task_id

if [ ! -z "$task_id" ]; then
    echo "=== 任务详情 ==="
    sqlite3 frontend/tasks.db -column -header "
    SELECT * FROM tasks WHERE id LIKE '$task_id%'"
fi
```

然后运行：
```bash
chmod +x view_db.sh
./view_db.sh
```

## 数据库结构

数据库文件位置：`frontend/tasks.db`

表结构 `tasks`:
- `id` - 任务ID (主键)
- `status` - 任务状态 (running/completed/error)
- `config` - 任务配置 (JSON格式)
- `created_time` - 创建时间
- `start_time` - 开始时间
- `end_time` - 结束时间
- `progress` - 进度 (0-100)
- `current_step` - 当前步骤
- `results` - 结果 (JSON格式)
- `error` - 错误信息
- `updated_at` - 更新时间

## 常用查询命令

### 查看最新任务
```bash
sqlite3 frontend/tasks.db "SELECT * FROM tasks ORDER BY created_time DESC LIMIT 1"
```

### 查看任务结果
```bash
sqlite3 frontend/tasks.db "SELECT results FROM tasks WHERE id='任务ID'"
```

### 查看任务配置
```bash
sqlite3 frontend/tasks.db "SELECT config FROM tasks WHERE id='任务ID'"
```

### 以 JSON 格式输出
```bash
sqlite3 frontend/tasks.db "SELECT json_object('id', id, 'status', status, 'config', config) FROM tasks WHERE id='任务ID'"
```

## 修复 VS Code SQLite 扩展问题

如果遇到 SQL 解析错误，可以尝试：

1. 更新扩展版本
2. 重新打开 VS Code
3. 使用 `sqlite3` 命令行工具（推荐）
